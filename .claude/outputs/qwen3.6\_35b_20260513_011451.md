# Task: Scrie logica de calcul pentru daysUntilExpiry pe modelul FoodItem cu expiryDate timestamp Firestore
**Model:** qwen3.6:35b
**Time:** 20260513_011451

Iată logica de calcul optimizată pentru Flutter + Firestore, cu atenție la fusuri orare, DST și comportamentul calendaristic:

### 🔹 Cod Dart (Model/Getter)

```dart
import 'package:cloud_firestore/cloud_firestore.dart';

class FoodItem {
  final String id;
  final Timestamp? expiryDate; // Firestore Timestamp (poate fi null)

  FoodItem({required this.id, this.expiryDate});

  /// Calculează zilele rămase până la expirare.
  /// - > 0 : zile rămase
  /// - 0   : expiră astăzi
  /// - < 0 : deja expirat
  int get daysUntilExpiry {
    if (expiryDate == null) return -1; // sau 0, în funcție de business logic

    // 1. Normalizare la fusul orar local al dispozitivului
    final now = DateTime.now().toLocal();
    final expiry = expiryDate!.toDate().toLocal();

    // 2. Tăiere la începutul zilei (ignorăm ora pentru comparație calendaristică)
    final nowDate = DateTime(now.year, now.month, now.day);
    final expiryDateOnly = DateTime(expiry.year, expiry.month, expiry.day);

    // 3. Diferența în zile calendaristice
    return expiryDateOnly.difference(nowDate).inDays;
  }

  /// Proprietăți auxiliare utile în UI
  bool get isExpired => daysUntilExpiry < 0;
  bool get isExpiringSoon => daysUntilExpiry >= 0 && daysUntilExpiry <= 3;
}
```

### 🔍 Explicații tehnice cheie

| Aspect | De ce contează | Soluția aplicată |
|--------|----------------|------------------|
| **Fus orar** | Firestore stochează UTC, `DateTime.now()` este local. Compararea directă poate da erori de ±1 zi. | `.toLocal()` pe ambele valori înainte de normalizare. |
| **DST (Daylight Saving Time)** | Trecerea la/oră de vară poate crea zile de 23h sau 25h, stricând `inDays`. | Normalizarea la `DateTime(year, month, day)` elimină efectul orelor. |
| **Truncare vs Rotunjire** | `difference.inDays` trunchiază spre zero. 23h = `0`, 24h = `1`. | Corect pentru "zile calendaristice". Dacă vrei să numără și parțialul ca `1`, folosește `ceil()`. |
| **Null Safety** | `expiryDate` poate fi `null` în Firestore. | Guardă explicită + valoare default `-1` (expirat/invalid). |

### 📱 Exemplu de utilizare în UI (Flutter)

```dart
Text(
  item.daysUntilExpiry > 0
      ? 'Expiră în ${item.daysUntilExpiry} zile'
      : item.isExpired
          ? '⚠️ Expirat'
          : 'Expiră astăzi',
  style: TextStyle(
    color: item.isExpiringSoon ? Colors.orange : Colors.green,
  ),
)
```

### ⚠️ Recomandări suplimentare
- Dacă vrei să incluzi **ora exactă** în calcul (ex: pentru notificări push), folosește direct `expiryDate!.toDate().difference(DateTime.now()).inDays` fără normalizare la zi.
- Pentru sincronizare offline, stochează `expiryDate` ca `Timestamp` în Firestore, nu ca string.
- Dacă ai nevoie de **zile complete** (ex: 23h = 1 zi), înlocuiește `.inDays` cu `.inDays + (difference.inMilliseconds > 0 ? 1 : 0)` sau `.ceil()`.
